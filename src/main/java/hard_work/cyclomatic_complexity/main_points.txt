Осн. вариант:
1) 10-15 инструкций функции
2) Нет switch/case, for, while, null
3) Избавляться от else, стараться избавлять от if
4) Разделять модули на открытые/закрытые
5) ad-hoc полимофризм. (перегрузка метода по аргументам одного и того же базового типа)
6) Астрактаный класс как автомат состояний, похоже на Null Object Pattern
7) Multiple Dispatch (в Java это как в п. 5)

Более мягкая версия снижения ЦС:
1. Запрещены else и любые цепочки else if.
2. Запрещены if, вложенные в if, и циклы, вложенные в if.
3. if внутри цикла допускается только один, и только для прерывания его работы (break/continue), выхода из функции (return) или генерации исключения.

1) Не определять методы, которые ничего не вычисляют (void)
